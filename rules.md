# Development Rules

## General

Use JavaScript.

Use functional components only.

Use React Hooks.

No class components.

---

## Components

Keep components reusable.

Maximum 250 lines per file.

Split large pages into smaller components.

---

## Styling

Tailwind CSS only.

No inline styles.

No CSS files unless global.

Use responsive layouts.

---

## Naming

PascalCase

Components

camelCase

Functions

kebab-case

Folders

UPPER_CASE

Constants

---

## Routing

Public Routes

/

Services

Provider Profile

Login

Register

Protected Routes

Customer Dashboard

Provider Dashboard

Admin Dashboard

---

## Authentication

Never store authentication manually.

Use Supabase Auth.

Protect dashboard routes.

---

## State

React Query for server state.

Context API for global state.

No Redux.

---

## Forms

React Hook Form

Zod validation

---

## Icons

Lucide React

---

## Code

Never duplicate code.

Always create reusable components.

Prefer composition over inheritance.

Always use TypeScript interfaces.

---

## UI

Responsive

Accessible

Dark Mode Ready

Loading States

Error States

Empty States

Animations

---

## Git

Meaningful commits.

Feature branches.

No direct commits to main.
